## What's wired

- `api/observability.py` — TracerProvider + helpers (`traced_span`, `trace_llm_call`)
- `api/main.py` — `setup_tracing(app)` on app startup; FastAPI auto-instrumentor wraps every HTTP request with a server span
- Manual spans inside the LLM-touching modules:
  - `core/regs.py` → `regs.retrieve_chunk` (records `cited_section`, `score`, `match`)
  - `core/reasoning.py` → `reasoning.chat` (records `zone_id`, `regulation_present`, `temperature`, `retry_directive`)
  - `core/guardian.py` → `guardian.energy_safety` and `guardian.regulation_consistency` (or `guardian.regulation_consistency.auto_pass` when null-citation path fires)
  - `core/pipeline.py` → `pipeline.process_zone` wraps each per-zone retry loop, so the trace tree shows zone → retrieval → reasoning → validator (no span; instant) → guardian-energy and guardian-reg in parallel

The result is a clean parent-child tree per HTTP request:

```
POST /api/sessions
└── pipeline.process_zone (×N zones, parallel)
    ├── regs.retrieve_chunk
    ├── reasoning.chat
    └── guardian.* (energy + reg in parallel)
```

## Run-by-run procedure (one-shot, ~10 min)

### 1. Start a local Jaeger all-in-one

```bash
docker run -d --rm --name jaeger \
  -p 16686:16686  \
  -p 4317:4317    \
  jaegertracing/all-in-one:latest
```

- Port `16686` — Jaeger UI (open http://localhost:16686)
- Port `4317` — OTLP gRPC receiver (where our OTel exporter sends)

If you don't have Docker handy, the alternative is `OVERRIDE_TRACING=console` which dumps spans as JSON to stdout — fine for verification, not photogenic for a screenshot.

### 2. Start uvicorn with tracing enabled

```bash
OVERRIDE_TRACING=otlp \
OTEL_EXPORTER_OTLP_ENDPOINT=http://localhost:4317 \
OTEL_SERVICE_NAME=override-api \
.venv/bin/uvicorn api.main:app --reload --port 8000
```

Look for the log lines:
```
setup_tracing: OTLP exporter → http://localhost:4317
setup_tracing: FastAPI auto-instrumentation active
```

### 3. Trigger a real pipeline run

Two options:

**a) Via the UI**

### 4. Capture the trace in Jaeger UI

Open http://localhost:16686. Service: `override-api`. Operation: `POST /api/sessions`. Click *Find Traces*.

In the trace detail view:
- The root span is the FastAPI server span (`POST /api/sessions`).
- 5 `pipeline.process_zone` children (parallel — staircase by start-time, overlap on duration).
- Each `pipeline.process_zone` has children: `regs.retrieve_chunk`, `reasoning.chat`, `guardian.energy_safety`, `guardian.regulation_consistency`.
- Hover any LLM span → see attributes (`zone_id`, `model_id`, `temperature`, `cited_section`, `score`, `override.duration_ms`).

Expand the trace; the visual tells the story of "5 zones reasoned in parallel, two-pass safety scored per zone." That's the demo asset.

## Troubleshooting

- **No traces appear in Jaeger UI**: confirm uvicorn log shows *"OTLP exporter → http://localhost:4317"*. Confirm port 4317 isn't blocked. `docker logs jaeger` should show no errors.
- **Spans appear but with empty attributes**: env var likely wasn't picked up. Verify `OVERRIDE_TRACING=otlp` in the same shell as uvicorn.
- **Pipeline runs but no manual spans, only the FastAPI request span**: import-order issue. Restart uvicorn — `api.observability` must be imported before any `core/*` module call.
- **OTLP gRPC connection refused** (uvicorn log): Jaeger may not be ready. `curl -s http://localhost:16686` should return HTML; if not, wait 3–5 seconds.
- **Trace screenshot looks bare with only one span**: trigger 2-3 sessions back-to-back; the screenshot looks better with multiple zones in flight.
