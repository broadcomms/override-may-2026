## Lessons (apply when implementing `core/regs.py`)

1. **`max_num_pages`** in `DocumentConverter.convert()` is the **document-size cap**, not "process the first N pages." Pass it ≥ the actual page count (or leave the giant default) — otherwise the converter rejects the document with `ConversionError("Input document ... is not valid.")`. Use `page_range=(start, end)` instead to select pages.

2. **Disable OCR** for FIA PDFs. They are text-based, not scanned. `PdfPipelineOptions.do_ocr = False` skips a ~40 MB model download (RapidOCR det/cls/rec) and saves significant time per page. First run with OCR enabled took 30+ minutes on a 207-page doc.

3. **Disable table structure** unless we explicitly need table layout. `PdfPipelineOptions.do_table_structure = False`. The energy-management section we care about is mostly prose + numbered articles.

4. **Use `PyPdfiumDocumentBackend`** rather than the default `DoclingParseV2DocumentBackend`. The default backend triggered "Input document is not valid" on both FIA PDFs even after the page-cap fix; PyPdfium opens them cleanly.

5. **HuggingFace deprecation warning is harmless.** "You are sending unauthenticated requests to the HF Hub" — just affects rate limits. Set `HF_TOKEN` in `.env` if pulls fail.

## Minimal kicker code (drop into `core/regs.py` skeleton at P2.5)

```python
from docling.document_converter import DocumentConverter, PdfFormatOption
from docling.datamodel.base_models import InputFormat
from docling.datamodel.pipeline_options import PdfPipelineOptions
from docling.backend.pypdfium2_backend import PyPdfiumDocumentBackend

def _converter() -> DocumentConverter:
    opts = PdfPipelineOptions()
    opts.do_ocr = False
    opts.do_table_structure = False
    return DocumentConverter(
        format_options={
            InputFormat.PDF: PdfFormatOption(
                pipeline_options=opts,
                backend=PyPdfiumDocumentBackend,
            )
        }
    )

def extract_pages(pdf_path: str, page_range=(1, None)) -> str:
    """Return Markdown for `page_range` of the PDF. (1, None) = whole document."""
    end = page_range[1] if page_range[1] is not None else 9_999
    result = _converter().convert(pdf_path, page_range=(page_range[0], end))
    return result.document.export_to_markdown()
```

## Energy-management section discovery

First 10 pages are TOC + general principles + definitions. The energy-management content lives later (Article 5 power unit, ~pages 30–60 in Issue 8 / further back in Issue 12). G-4's discovery step:

1. Extract the full document Markdown
2. Locate the section heading via heuristic (e.g., regex on `^##\s+ARTICLE\s+5` or text matching against "Power Unit")
3. Slice that section's chars
4. Chunk into ≤ 1000-char `RegulationChunk` instances
5. Embed via `ibm/granite-embedding-278m-multilingual`
6. Persist to `data/regs/extracted_chunks.sample.json`

The exact `section` string (e.g., `"5.4.2"`) is read out of the Docling extraction at runtime — never hardcoded.
