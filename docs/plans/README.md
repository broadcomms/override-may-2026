# plans/

Persistent planning markdown — survives across conversations. (Session pattern.)

## Convention

- One file per feature: `docs/plans/{feature-slug}.md`.
- Bob in **Plan mode** writes to this folder.
- Bob in **Code mode** reads from this folder via `@docs/plans/{slug}.md`.
- After a feature's implementation is committed, **mark the plan file in the same PR as complete** (Session pattern).

## Forthcoming plans

- `{feature-slug}.md` — Pending feature plan example file in order of implementation.
